# Contract: Submission Package

## Purpose

Define the non-code deliverables required before the app can be submitted for
Shopify App Store review.

## Required Package Elements

### Listing identity contract

- App name, icon, and merchant-facing positioning must be consistent across the
  Partner Dashboard, app listing, and in-app experience.
- Pricing information must appear only in approved pricing sections and must
  match the live billing experience.

### Media contract

- Screenshots must show real product surfaces and unique states of the app.
- Screencast must demonstrate onboarding, the core workflow, and the review path
  in English or with English subtitles.

### Credential contract

- Reviewer credentials must be valid, current, and grant access to the complete
  feature set needed for review.
- Test data and sample files must be sufficient to complete the documented
  workflow without extra setup.

### Instruction contract

- Testing instructions must describe the exact install, billing, workflow, and
  verification path the reviewer should follow.
- Any geographic, permission, or environment prerequisites must be stated
  explicitly before review starts.

### Support contract

- Emergency/support contact details must be current before submission.
- The review package must identify how Shopify can reach the owner if a critical
  issue is found during review.
